# 05 — Internacjonalizacja (i18n)

## Setup next-intl

### Instalacja

```bash
npm install next-intl
```

### `lib/i18n/routing.ts`

```typescript
import { defineRouting } from 'next-intl/routing'
import { createNavigation } from 'next-intl/navigation'

export const routing = defineRouting({
  locales: ['pl', 'en'],
  defaultLocale: 'pl',
  localePrefix: 'always', // /pl/... i /en/... zawsze explicit
})

export const { Link, redirect, usePathname, useRouter, getPathname } =
  createNavigation(routing)
```

### `lib/i18n/request.ts`

```typescript
import { hasLocale } from 'next-intl'
import { getRequestConfig } from 'next-intl/server'

import { routing } from './routing'

export default getRequestConfig(async ({ requestLocale }) => {
  const requested = await requestLocale
  const locale = hasLocale(routing.locales, requested) ? requested : routing.defaultLocale

  return {
    locale,
    messages: (await import(`./messages/${locale}.json`)).default,
  }
})
```

### `next.config.ts`

```typescript
import createNextIntlPlugin from 'next-intl/plugin'

const withNextIntl = createNextIntlPlugin('./lib/i18n/request.ts')

const config = {
  // ...reszta konfiguracji
}

export default withNextIntl(config)
```

## Użycie w komponentach

### Server component

```typescript
import { getTranslations } from 'next-intl/server'

export default async function TripsPage() {
  const t = await getTranslations('trips')
  return <h1>{t('title')}</h1>
}
```

### Client component

```typescript
'use client'
import { useTranslations } from 'next-intl'

export function TripCard() {
  const t = useTranslations('trips.card')
  return <span>{t('countries', { from: 'PL', to: 'HU' })}</span>
}
```

## LocaleSwitcher component

```typescript
'use client'
import { useLocale } from 'next-intl'
import { usePathname, useRouter } from '@/lib/i18n/routing'

export function LocaleSwitcher() {
  const locale = useLocale()
  const router = useRouter()
  const pathname = usePathname()

  const switchTo = (newLocale: 'pl' | 'en') => {
    router.replace(pathname, { locale: newLocale })
    // Update profile preference async
    // (hook useProfile dostaje updateLocale)
  }

  return (
    <Select value={locale} onValueChange={(v) => switchTo(v as 'pl' | 'en')}>
      <SelectItem value="pl">Polski</SelectItem>
      <SelectItem value="en">English</SelectItem>
    </Select>
  )
}
```

Przy zmianie locale:
1. Zmień URL (`router.replace(pathname, { locale })`)
2. Zaktualizuj `profiles.preferred_language` (async, nie blokuje UI)
3. Przy kolejnym logowaniu — czytaj preferowany locale z profilu, redirect jeśli inny niż aktualny URL

## Format dat i walut

Zawsze używaj `next-intl` formatters + `Intl`:

```typescript
import { useFormatter, useLocale } from 'next-intl'

export function Date({ value }: { value: Date }) {
  const format = useFormatter()
  return format.dateTime(value, { dateStyle: 'medium' })
}

// Dla walut lepiej lib/utils/format.ts z helperem:
// formatCurrency(amount, currency, locale) → używa Intl.NumberFormat
```

## Fallback

Jeśli brakuje klucza w `en.json`, `next-intl` pokazuje klucz w UI w dev mode. W produkcji — upewnij się, że oba pliki są zsynchronizowane (można zrobić prosty test w CI który sprawdza symetrię).
